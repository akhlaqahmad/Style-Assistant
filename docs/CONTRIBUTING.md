# Contributing to StylistA

We welcome contributions to the StylistA project!

## Code of Conduct

This project adheres to the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). By participating, you are expected to uphold this code.

## Getting Started

1.  **Fork the repository**.
2.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/your-feature-name
    ```
3.  **Make your changes**.
4.  **Test your changes** thoroughly.
5.  **Commit your changes** with descriptive commit messages:
    ```bash
    git commit -m "feat: Add new outfit generator feature"
    ```
6.  **Push your branch** to your fork:
    ```bash
    git push origin feature/your-feature-name
    ```
7.  **Submit a Pull Request** to the `main` branch.

## Development Workflow

1.  **Follow Coding Standards**: Use consistent formatting and follow established patterns.
2.  **Write Tests**: Add unit tests or integration tests for new features.
3.  **Document Changes**: Update documentation (README, docs/) if necessary.
4.  **Review Code**: Before submitting a PR, review your own code for potential issues.

## Reporting Bugs

Please report bugs by opening an issue on GitHub. Include:

-   Description of the issue.
-   Steps to reproduce.
-   Expected behavior vs. actual behavior.
-   Environment details (OS, device, Expo version).

## Feature Requests

We welcome feature suggestions! Open an issue describing the feature and its benefits.
